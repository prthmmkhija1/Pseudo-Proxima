# Contributing

## Workflow
1. Fork and create a feature branch
2. Add tests for changes
3. Run `pytest` and lint checks
4. Submit a pull request with summary and screenshots if UI changes

## Coding standards
- Follow PEP 8
- Prefer type hints
- Keep functions small and focused

## Commit messages
- Use imperative mood, e.g., "Add qiskit adapter timeout"

## Code review
- Address comments promptly
- Add tests for bug fixes
